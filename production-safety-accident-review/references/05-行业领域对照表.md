# 行业领域判定与对照表（第五步）

来源：《生产安全事故统计调查制度》附录（二）"管理分类与国民经济行业分类（GB/T 4754-2017）（2019年修订）对照表"。本文件收录**全部六个管理分类**（A / B / C·F·H / E / G / D·I—T），所有"不含""涉及……活动的"限定均照原文保留。

> **判定基准（《统计小助手判定规则》第五节）**：一起事故的行业领域，要**同时给出①管理分类的行业领域 + ②国民经济行业分类的行业领域**，二者**一一对应**；依据即本对照表（附录二）+《2017国民经济行业分类注释》。因此第五步**必须两个分类都输出**（管理分类 + 国民经济行业四位小类码及其名称），不能只给其一；能定到唯一行业时二者成对给出、相互印证。**注意两套分类要分两行各自写全，不要混**：管理分类链止于它自身的末级（如"C·F·H 商贸制造业·工贸·机械"），**不要把国民经济行业的大类名（如"汽车制造业"）并进管理分类链**；国民经济行业分类单独一行按"门类→大类→中类→小类"写全。

## 判定与编码步骤

1. **先定"事故发生单位"**：按归属规则（建筑业记施工单位、外包矿山工程记发包单位、租赁/分包按核算关系、设备不合格记使用单位等，见 `references/02` 第13—20条）。行业按**该单位所属行业**判，不是按现场活动想当然。
2. **判该单位所属行业——先看纳统情形自带的行业，再从事件经过反推，公司名交叉核对，"待核实"最后**：
   - **⓪ 纳统情形自带的行业归属（第二步命中即最优先采用）**：很多纳统情形本身就把事故归到特定行业——此时**行业由该情形/事故类型直接确定，可以不同于单位工商主营**，据此定领域，再据经过细化到四位小类：
     - **十二类道路运输车辆**（含企业通勤车）在运输活动中 → **G 交通运输业·道路运输业(54)**（"道路运输事故"统计）
     - **建筑施工 / 城乡新建、改建、修缮、拆除建筑**过程 → **E 建筑业**（记施工单位）
     - **煤矿 / 金属非金属矿山 / 石油天然气开采外包工程** → **B 采矿业**（煤矿06 / 金属非金属矿08·09·10 / 油气07，记发包单位）
     - **人员密集场所**：商场→零售业(52)、宾馆→住宿业(61)、餐饮→餐饮业(62)、歌舞厅·网吧·景区→文化体育娱乐业(R)等
     - **渔船作业**→A·渔业船舶；**农机作业**→A·农业机械；**水上运输**→G·水上运输业(55)
     - 命中⓪则以⓪定领域；未命中再走①②。（即：第二步"是否纳统/命中哪条情形"的结论，要带到第五步用来定行业。）
   - ① **从事件经过抓行业指示词（主信号，必做且要展示过程）**：通读**整段警情**，提取能指示行业的关键词——**产品/原料、工艺、专用设备、场所类型、作业类型**（如冶炼、焊接装配、印染、光伏安装、矿井开采、货物运输、餐饮后厨、窑炉、货架仓储等）。据此反推行业，给出候选（哪怕只落到管理分类或大类）。
     - **通用设备/动作：不能单独定行业，但能缩小候选**——升降平台、起重、行车、叉车、货车、脚手架、电焊机、装卸搬运等多行业通用，不足以定到唯一行业；**但"起重/升降/装卸/搬运重物料"这类信息指向"制造业(C)/仓储物流/建筑业(E)"一类实体重工行业**，可据此列候选、排除金融/信息/农业等明显不相关的，再结合产品/工艺/场所或补充信息收窄。**事故经过里每个关键词都要过一遍、榨干信息，不要只看公司名。**
     - **防背离**：活动≠单位主营时（如食品厂员工临时修厂房摔伤），按单位主营（食品制造）判；但建筑施工、农机作业、渔船作业、运输活动等，本对照表按"涉及…活动的"落域，此时活动即判据。
   - ② **公司名交叉核对（仅辅助，绝不单独定案）**：`python3 scripts/classify_org.py --name "<单位全称>"` 给行业候选（启发式、仅参考）。**无论公司名多明确，都不得单独作为最终行业答案——必须回到事故经过（①）印证、两方结合定案**：
     - ①②一致 → 高置信，定唯一行业；
     - 仅①有 → 据①；
     - 仅②有（事故经过确无行业线索）→ 只能作**低置信候选**，须注明"仅据名称推测、事故经过未印证，待核实"，**不得当确定结论**；
     - ①②矛盾 → **以①（事故经过）+ 本对照表为准**，名字让位。
     - 它的"待核实"不得否定①的推断。
     - **该脚本是内部工具**：最终研判里**不要出现脚本名或"启发式归类"字样**，只用自然语言写结论（如"单位名称无行业指示"），实现细节不给用户看。
   - ③ **企业库/在线兜底**：暂无（待数据）。
   - **信号指向多个可能行业时——把候选都列出来，不要直接"待核实"**：据事件+名称能缩小到几个合理行业的（如通用设备"简易升降平台"常见于 制造业(C)、仓储物流、建筑业(E) 等），就把这几个**候选行业全部输出**，按可能性排序、标注各自的管理分类/大类；因尚未定到唯一行业，WSA 编码可分别给各候选的码、或注明"待定到唯一行业后出码"。同时提示："具体属哪一行需补充单位主营（营业执照经营范围／实际产品或作业）确认"。候选须**受证据约束、合理有界**——只列事件/名称支持的那几个，不要罗列全部六大门类。
     > 「单位主营」这一栏能不能填，见 `references/00` 的**来源限定**：无逐字出处即填"未掌握"。**主营"未掌握"正是本条"列候选"的触发条件**，不是"无法判断"——两者别混。
   - **只有连"可能行业候选"都列不出（事件与名称完全无任何行业指示）时**，才标 **"无法判断，待补充单位主营信息"**，并写清已从事件与名称两方面查找过。
3. 按下表把行业对应到**管理分类**（务必尊重每条的"不含""涉及……活动的"限定）。
4. 细化到 **GB/T 4754-2017 四位小类代码**（用 `scripts/encode.py --search 关键词` 查候选，再据《2017国民经济行业分类注释》确认）。
5. 用 `scripts/encode.py --code <四位码>` 生成 **WSA3XXXX** 编码——位数由脚本保证，不手写。

> 表中给到大类（2位）或中类（3位）的，表示该范围整体归入某管理分类；WSA 编码仍需落到范围内的具体**四位小类**。
> 当前工作重点为 B 采矿业、C·F·H 商贸制造业，但对照表与编码对全部行业均可用，其余管理分类同样正常判定并编码。

# 事故等级判定依据（第三步）

来源：《生产安全事故报告和调查处理条例》（国务院令第493号）第三条；与《生产安全事故统计调查制度》"四、主要指标解释"第9条完全一致。

判定方式：由脚本 `scripts/grade.py` 精确计算，不由模型推断。模型只负责从警情文本抽取"死亡人数、重伤人数、直接经济损失"三个数值。

## 阈值表

| 等级 | 死亡人数 | 重伤人数（含急性工业中毒） | 直接经济损失 |
|---|---|---|---|
| 特别重大 | 30人以上 | 100人以上 | 1亿元以上 |
| 重大 | 10人以上30人以下 | 50人以上100人以下 | 5000万元以上1亿元以下 |
| 较大 | 3人以上10人以下 | 10人以上50人以下 | 1000万元以上5000万元以下 |
| 一般 | 3人以下 | 10人以下 | 1000万元以下 |

## 计算规则（务必遵守）

1. **取最高**：死亡、重伤、直接经济损失三个维度分别定级，最终等级取三者中最高的一级。
2. **"以上"含本数，"以下"不含本数**（条例原文）。例如死亡恰好3人属"较大"（不属"一般"）；恰好10人属"重大"。
3. **单位换算**：直接经济损失以万元计，1亿元=10000万元。
4. **口径对齐**：
   - "死亡人数"含下落不明人数；下落不明满30日按死亡统计（火灾、道路运输另有时限规定）。
   - "重伤人数"含急性工业中毒人数（统计制度一般规则第21条：急性工业中毒计为重伤）。
5. **本步只定等级，不决定是否纳入统计**。"无人员伤亡且直接经济损失小于100万元不纳入统计"是第二步（纳统判定）的内容，不在本步处理。

## 未掌握维度的处理（重要）

警情文本常常不给全三个维度的数字，**尤其是直接经济损失**。此时必须区分：

- **已知为0**：确有信息表明该维度为0（如"无人受伤"），传入 `0`。
- **未掌握**：文本没说、或只描述了后果未量化（如"厂房烧毁""设备损毁"却无金额），传入 `None`（脚本命令行即"不传该参数"）。**绝不可用0冒充未掌握的损失**，否则会低估等级。

脚本按已知维度定级，未掌握维度只作"补充后可能上调"的提示。原理：未知维度只会让严重程度上升、不会下降，所以已知维度得到的等级是**下界**，补充后只可能上调、不会下调。

调用示例：
```
python3 scripts/grade.py --deaths 1                 # 仅知死亡1人，重伤/损失未掌握 → 一般(初判)
python3 scripts/grade.py --deaths 0 --serious 0 --loss 2000   # 损失2000万独立定级 → 较大
```

输出的"研判说明"须如实标注未掌握维度及"等级为初判、补充后可能上调"，保证可追溯。
